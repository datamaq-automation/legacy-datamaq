# Auditoría técnica inicial — profebustos-www (17 feb 2025)

## Resumen ejecutivo
- La base Vue 3 + Vite está ordenada, pero conviene reforzar tipado, nomenclatura y cobertura de pruebas para sostener la escalabilidad.
- Hay buenas prácticas de accesibilidad (skip link, estados deshabilitados), aunque faltan avisos dinámicos y chequeos de contraste.
- La configuración de entornos debe endurecerse: validar URLs HTTPS y aislar plugins solo-desarrollo para evitar filtraciones en producción.
- Las imágenes sin optimizar y la inclusión de devtools en el bundle amenazan los objetivos de performance (NFR-031/NFR-032).
- La trazabilidad analítica está bien diseñada, pero requiere QA para evitar eventos duplicados y documentar contratos de datos.
- No existen automatizaciones de QA/CI ni documentación legal/compliance suficiente para la siguiente fase.

## 1. Calidad de código y arquitectura
- **Nomenclatura inconsistente**: el componente `SobreProfeBustos.vue` se importa como `Sobreprofebustos`, lo que complica la búsqueda y rompe la convención PascalCase/UpperCamelCase. Ajustar nombres y comentarios para coherencia global.【F:src/App.vue†L7-L16】【F:src/components/SobreProfeBustos.vue†L1-L28】
- **Tipado laxo**: `tsconfig.json` no habilita `strict`, `noImplicitAny`, ni `skipLibCheck`, reduciendo la capacidad del compilador para detectar errores tempranos. Activar el modo estricto y solventar avisos resultantes.【F:tsconfig.json†L1-L13】
- **Sin pruebas automatizadas**: `package.json` no define scripts ni dependencias de testing. Incorporar Vitest + Vue Test Utils para cubrir componentes críticos (CTA, formularios, servicios).【F:package.json†L1-L27】
- **Plugins de producción**: `vite.config.js` registra `vueDevTools()` sin condicionar al entorno, exponiendo tooling en builds productivos. Envolver en `if (process.env.NODE_ENV !== 'production')` o usar `mode` para excluirlo en producción.【F:vite.config.js†L1-L22】

## 2. Accesibilidad (WCAG 2.1 AA)
- **Feedback no anunciado**: el formulario muestra mensajes de éxito/error con `<p role="alert">`, pero no declara `aria-live`, lo que puede omitir anuncios en algunos lectores. Añadir contenedor con `aria-live="polite"` o `assertive`.【F:src/components/ContactFormSection.vue†L64-L118】
- **Foco post-envío**: tras enviar con éxito se limpia el formulario pero no se devuelve el foco a un aviso o encabezado, dificultando a usuarios de teclado saber qué ocurrió. Considerar `focus()` sobre el mensaje o encabezado dentro de `handleSubmit`.【F:src/components/ContactFormSection.vue†L90-L118】
- **Contraste y estados**: varios botones usan clases Bootstrap personalizadas (`bg-success-subtle`, `text-secondary`). Validar contraste mínimo 4.5:1, especialmente en tarjetas y badges del hero/servicios.【F:src/components/HeroSection.vue†L6-L79】【F:src/components/ServiciosSection.vue†L5-L95】

## 3. Seguridad
- **Validación de endpoints**: `config.ts` asegura emails y URLs HTTPS para `VITE_CHAT_URL`, pero `VITE_CONTACT_API_URL` solo se normaliza. Forzar `https://` mediante `ensureHttpsUrl` para cumplir NFR-021 y evitar llamadas a endpoints inseguros.【F:src/infrastructure/config.ts†L1-L63】
- **Exposición de datos sensibles**: `sendWhatsappContactEvent` envía `user_agent` y ubicación de página a la API. Documentar en la política de privacidad y confirmar que el backend trata estos datos bajo Ley 25.326.【F:src/appLogic.ts†L35-L88】
- **Dependencias críticas**: revisar versiones de `@microsoft/clarity` y `vue-gtag-next` frente a avisos de seguridad; automatizar con `npm audit` en CI.【F:package.json†L12-L26】

## 4. Rendimiento
- **Imágenes pesadas**: los assets principales pesan 1.7–2.7 MB cada uno. Convertir a WebP/AVIF, generar variantes responsivas y aplicar `srcset`/`sizes` para móviles.【F:src/components/HeroSection.vue†L41-L65】【F:src/components/ServiciosSection.vue†L12-L80】
- **Bundle inflado**: al incluir `vueDevTools` en producción y cargar `bootstrap/dist/css` completa, el `first load JS/CSS` crece. Evaluar `bootstrap/scss` con tree-shaking o `@vueuse/headlessui` para componentes puntuales.【F:src/main.ts†L1-L10】【F:vite.config.js†L9-L20】
- **Lazy-loading**: las secciones hero/servicios/contacto se montan juntas. Implementar lazy components (`defineAsyncComponent`) o splitting por ruta cuando se planifique multipágina para cumplir NFR-031.【F:src/App.vue†L6-L38】

## 5. Analítica y medición
- **Contrato de datosLayer**: `recordWhatsappEngagement` y `recordEmailEngagement` empujan a `window.dataLayer` y llaman `window.gtag`/`window.clarity` sin tipado global. Documentar la forma del objeto y añadir declaración `global.d.ts` para evitar regresiones al refactor.【F:src/application/services/analyticsTracker.ts†L1-L56】
- **Eventos duplicados**: `openWhatsApp` dispara `sendWhatsappContactEvent` (API) y `recordWhatsappEngagement` (analytics). Verificar en QA que GA4 y Clarity no reciban eventos repetidos por listeners adicionales (ej. etiquetado automático).【F:src/appLogic.ts†L59-L88】
- **Chat URL**: `getChatUrl()` devuelve string vacío cuando no está configurado; documentar comportamiento para que marketing active/desactive CTA sin cambios de código.【F:src/application/services/chatChannelService.ts†L1-L15】

## 6. Escalabilidad y mantenibilidad
- **Monolito de componentes**: `App.vue` ensambla todas las secciones; al crecer (BR-002) conviene mover contenido a rutas o módulos de contenido para multipágina. Evaluar `vue-router` y layout components.【F:src/App.vue†L6-L38】
- **Documentación desactualizada**: README menciona backend PHP/MySQL inexistente en repo. Actualizar con arquitectura real (API externa) y procedimientos de despliegue CI/CD.【F:README.md†L1-L46】
- **Convención de carpetas**: mezclar `application`, `infrastructure` y componentes es positivo, pero falta guía en docs para nuevos colaboradores. Añadir secciones en `docs/` explicando capas y responsabilidades.【F:src/application/services/emailChannelService.ts†L1-L39】【F:src/infrastructure/config.ts†L1-L63】

## 7. Cumplimiento legal
- **Avisos mínimos**: `LegalSection` resume restricciones pero no incluye política de privacidad ni aviso de cookies. Agregar enlaces a documentos legales una vez aprobados y considerar banner si se amplían rastreadores.【F:src/components/LegalSection.vue†L1-L11】
- **Gestión de consentimiento**: Clarity y GA4 se inicializan automáticamente al montar la app, sin solicitar consentimiento explícito. Evaluar implementación de CMP o banner conforme a Ley 25.326 y guías AAIP.【F:src/infrastructure/analytics.ts†L1-L26】

## 8. DevOps y despliegue
- **Sin CI/CD**: no hay workflows en `.github/`. Configurar GitHub Actions para ejecutar lint/test/build y auditorías (`npm run build`, `npm audit`).【F:package.json†L6-L21】
- **Backups/rollback**: documentar estrategia (por ejemplo, artefactos versionados + releases etiquetados). Actualmente no hay referencia en repo.【F:docs/srs-profebustos-www.md†L1-L120】
- **Variables de entorno**: crear plantilla `.env.example` con valores ficticios y aclarar su uso en despliegues automatizados.【F:src/infrastructure/config.ts†L33-L63】
